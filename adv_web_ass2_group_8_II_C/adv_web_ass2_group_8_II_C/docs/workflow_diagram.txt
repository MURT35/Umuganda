# Workflow Diagram — Umuganda Smart Service Request Platform

## System Flow

```
┌─────────────────────────────────────────────────────────────────┐
│                    PUBLIC USER (Resident/Student)               │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                    Visits Homepage (/)
                               │
                    Clicks "Submit Request"
                               │
               ┌───────────────▼───────────────┐
               │     Request Submission Form    │
               │  - Full Name                  │
               │  - Email                      │
               │  - Category                   │
               │  - Priority (Low/Med/High)    │
               │  - Description                │
               │  - JS Live Preview button     │
               └───────────────┬───────────────┘
                               │
                    JS Validates form (client)
                               │
                    PHP Validates (server)
                               │
               ┌───────────────▼───────────────┐
               │    RequestController.php       │
               │    calls Request model         │
               └───────────────┬───────────────┘
                               │
               ┌───────────────▼───────────────┐
               │    MySQL DB: requests table    │
               │    Status set to "Pending"     │
               └───────────────┬───────────────┘
                               │
                     Success page shown
                               │
═══════════════════════════════════════════════════════════════════
                    ADMIN FLOW
═══════════════════════════════════════════════════════════════════
                               │
               ┌───────────────▼───────────────┐
               │         Admin Login            │
               │   login.php → AuthController  │
               │   Session started if valid    │
               └───────────────┬───────────────┘
                               │
               ┌───────────────▼───────────────┐
               │       Admin Dashboard         │
               │  - Summary stats (4 cards)   │
               │  - Filter by status/category │
               │  - Table of all requests     │
               └───────────────┬───────────────┘
                               │
               Admin clicks "Accept" or "Resolve"
                               │
               ┌───────────────▼───────────────┐
               │     update_status.php          │
               │     Updates DB status          │
               │                               │
               │  Pending → In Progress        │
               │  In Progress → Resolved       │
               └───────────────────────────────┘
```

## Status Lifecycle

  [Pending] ──── Accept ───► [In Progress] ──── Resolve ───► [Resolved]

## MVC Structure

  public/index.php  (Entry point)
       │
       ├── app/controllers/   (Business logic, routing)
       ├── app/models/        (Database queries)
       ├── app/views/         (HTML output)
       ├── config/db.php      (DB connection)
       ├── assets/css/        (Styling)
       ├── assets/js/         (Live preview + validation)
       └── database/schema.sql (DB structure)
